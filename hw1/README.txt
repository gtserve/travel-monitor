> Compilation:
    $ make compile

> Execution:
    $ ./vaccineMonitor -c <recordFile> -b <bloomSize>

> Run bash script:
    $ bash ./testFile.sh <virusesFile> <countriesFile> <numOfLines> <duplicatesAllowed>


> Bash script:
    - Παράγονται IDs μόνο από 0 έως 9999
    - Ονόματα και επώνυμα διαλέγονται τυχαία από τα αντίστοιχα αρχεία στον φάκελο resources.
    - Ημερομηνίες παράγονται στο format DD-MM-YYYY.
    - Ελάχιστη ημερομηνία που μπορεί να παραχθεί είναι 01-01-1796 και μέγιστη 30-12-2021.
    - Αν οι ζητούμενες εγγραφές είναι πάνω από 10000, ανεξάρτητα από το duplicatesAllowed flag,
        παράγονται διπλότυπα IDs.
    - Οι διπλότυπες εγγραφές έχουν σχεδόν πάντα διαφορετικά τα υπόλοιπα χαρακτηριστικά εκτός από το
    ID. Γενικά το script παράγει σχεδόν αποκλειστικά μόνο αυτού του είδους inconsistency, δηλ.
    εγγραφές με ίδιο ID αλλά διαφορετικά τα υπόλοιπα στοιχεία.

> Δομές Δεδομένων:
    - Το SkipList υλοποιήθηκε με τέτοιο τρόπο ώστε να κρατάει οποιουδήποτε είδους αντικείμενο. Τα
        κλειδιά είναι πάντα ακέραιοι.


> Έξτρα Δομές Δεδομένων:
    - Υλοποίηθηκε HashTable με τέτοιο τρόπο ώστε να μπορεί να αντιστοιχίσει οποιουδήποτε είδους
    κλειδί σε οποιουδήποτε είδους αντικείμενο, το οποίο χρησιμοποιήθηκε ως εξής:
        - Αποθήκευση αντιστοίχισης Citizen ID -> Citizen
        - Αποθήκευση αντιστοίχισης Country Name -> Country
        - Αποθήκευση αντιστοίχισης Virus Name -> Virus
        - Aggregation των εμβολιασμών ανά χώρα για τις λειτουργίες /populationStatus και
            /popStatusByAge
    - Για κάθε ιό δημιουργείται μια 3η SkipList η οποία ταξινομεί βάσει σύνθετου κλειδίου, πρώτα με
        ημερομηνία και μετά με ID, πολίτες. Αυτή η δομή μας επιτρέπει να εκτελούμε τις λειτουργίες
        /populationStatus και /popStatusByAge με καλή πολυπλοκότητα.
